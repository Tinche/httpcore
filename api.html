Async API Overview¶

Base async interfaces¶

These classes provide the base interface which transport classes need to implement.

class httpcore.AsyncHTTPTransport[source]¶

The base interface for sending HTTP requests.

Concrete implementations should subclass this class, and implement the handle_async_request() method, and optionally the aclose() method.

async handle_async_request(method, url, headers, stream, extensions)[source]¶

The interface for sending a single HTTP request, and returning a response.

Parameters

  • method (bytes) – The HTTP method, such as b'GET'.

  • url (Tuple[bytes, bytes, Optional[int], bytes]) – The URL as a 4-tuple of (scheme, host, port, path).

  • headers (List[Tuple[bytes, bytes]]) – Any HTTP headers to send with the request.

  • stream (httpcore.AsyncByteStream) – The body of the HTTP request.

  • extensions (dict) – A dictionary of optional extensions.

Returns

  • status_code – The HTTP status code, such as 200.

  • headers – Any HTTP headers included on the response.

  • stream – The body of the HTTP response.

  • extensions – A dictionary of optional extensions.

Return type

Tuple[int, List[Tuple[bytes, bytes]], httpcore.AsyncByteStream, dict]

async aclose()[source]¶

Close the implementation, which should close any outstanding response streams, and any keep alive connections.

Return type

None

Async connection pool¶

class httpcore.AsyncConnectionPool(ssl_context=None, max_connections=None, max_keepalive_connections=None, keepalive_expiry=None, http1=True, http2=False, uds=None, local_address=None, retries=0, max_keepalive=None, backend='auto')[source]¶

Bases: httpcore.AsyncHTTPTransport

A connection pool for making HTTP requests.

Parameters

  • ssl_context (ssl.SSLContext) – An SSL context to use for verifying connections.

  • max_connections (int) – The maximum number of concurrent connections to allow.

  • max_keepalive_connections (int) – The maximum number of connections to allow before closing keep-alive connections.

  • keepalive_expiry (float) – The maximum time to allow before closing a keep-alive connection.

  • http1 (bool) – Enable/Disable HTTP/1.1 support. Defaults to True.

  • http2 (bool) – Enable/Disable HTTP/2 support. Defaults to False.

  • uds (str) – Path to a Unix Domain Socket to use instead of TCP sockets.

  • local_address (str) – Local address to connect from. Can also be used to connect using a particular address family. Using local_address="0.0.0.0" will connect using an AF_INET address (IPv4), while using local_address="::" will connect using an AF_INET6 address (IPv6).

  • retries (int) – The maximum number of retries when trying to establish a connection.

  • backend (Union[httpcore._backends.base.AsyncBackend, str]) – A name indicating which concurrency backend to use.

  • max_keepalive (int) –
